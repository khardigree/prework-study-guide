HTML

  • HTML is responsible for the "what" that is in a website. The words, images, and layout are all HTML.
  • The head element contains info about the webpage. This is invisible to the user.
  • The meta element contains info about the webpage, and is what is housed in the head element listed above. This is also invisible to the user.
  • The body element conatins the information that is presented to the user. This is the "website" that the user interacts with.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.
  • CSS stands for Cascading Style Sheets
  • Inline CSS is used when you want style elements directly in the HTML file. You apply the CSS directly to specific HTML tags. This is useful for quick, permenant changes, but can be less felxible due to editing needing to happen on an individual basis.
  • Internal Style Sheets apply CCS to the whole page, but only that page. To use, you put the style designation in the head element. A single change to the CSS rule changes all tagged elements.
  • External CSS Style Sheets are the most common method of applying CSS to a webpage. To use, you link a seperate file. This adheres to the Seperation of Concerns coding principle: every section of code has its own responsibility.

Git

  • Git is a version control system that allows developers to store their code, work from different computers, work in tandem with other developers without overlap, track versions, projects, etc.
  • GitHub is a webpage that almost acts as a social media platform for sharing and viewing projects, as well as a repository system for coding projects, provides a workflow system, etc for developers.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: switches you to the main branch
  • git add -A: stage your changes
  • git commit -m "note regarding changes": commits files to working branch

JavaScript

  • JavaScript is a high-level dynamic programming language that adds interactivity to websites.
  • A variable is a named container that allows you to store data in your code.
  • const: allows you to create a variable, but you cannot reassign the data. This is the preferred method of naming variables.
  • if: introduces decision making into code. Only executes code if the statement meets a condition, or is true.
  • falsy: 0, -0, " ", null, undefined, NaN, or a Boolean with a value of false.
  • truthy: all other values.
  • === called the strict equality operator, checks to see if two values are equal, returns either true or false.
  • array: a single variable that is used to hold a group of data, generally related.
  • index: to get to one item in an array, an index is corrected. Always starts with 0.
  • for: uses predictable patterns in indices to perform a task on all items in an array by allowing a single code block to be executed over and over.
  • let: assign data to a variable, but unlike cont, allows value reassignment over time.
  • function: a set of instructions that tells the computer how to perform a certain task. Functions do not automatically run, but are helpful because you can call them when you need them.
